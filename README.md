# Cartridge Valve Document Management System

一个基于 **Django** 的插装阀技术文档管理与查询系统，  
用于企业内部对插装阀产品及其相关技术文档（PDF）进行统一管理和展示。

---

## 📌 项目背景

在插装阀相关的工程应用中，不同型号的产品通常会对应多份技术文档，
同时也存在 **多个产品共用同一份文档** 的情况。

传统以文件夹形式管理 PDF 文档存在以下问题：

- 文档分散，不易查询
- 产品与文档关系混乱
- 无权限控制，信息安全性不足

本项目旨在通过 Web 系统的方式，实现 **产品信息与技术文档的结构化管理**，
提升内部资料的可维护性与检索效率。

---

## 🛠 技术栈

- **后端框架**：Django 4.2
- **编程语言**：Python 3.8+
- **数据库**：SQLite（开发阶段）
- **前端模板**：Django Templates
- **文件存储**：Django Media Files
- **权限系统**：Django Authentication
- **版本控制**：Git / GitHub

---

## ✨ 核心功能

### 🔹 产品管理
- 插装阀产品列表展示
- 产品详情页查看
- 支持产品描述与基础信息维护

### 🔹 文档管理
- 技术文档（PDF）上传
- 文档与产品 **多对多关联**
- 文档列表展示
- 支持在线预览与下载

### 🔹 权限控制
- 基于 Django 用户系统
- 未登录用户无法访问技术文档
- 登录后可查看、预览和下载文档

---

## 🧱 数据模型设计（简述）

- **Product（插装阀）**
- **Document（技术文档）**

两者之间采用 **多对多关系**：
- 一个产品可关联多个文档
- 一个文档可被多个产品共用

该设计更符合真实工程场景，具有良好的扩展性。
